# Jungle Tank

탑뷰 시점에서 탱크를 조작하는 싱글플레이 웹 게임입니다.  
현재 범위는 `플레이어 1명 + 아군 봇 vs 적 봇` 구조의 팀전 MVP이며, 실시간 멀티플레이는 아직 포함하지 않습니다.

## 핵심 기능

- HTML/CSS/Vanilla JS Canvas 기반 탱크 전투
- 메뉴에서 아군/적군 수를 조절하는 `N vs M` 팀전 설정
- 5개 맵 선택 UI
- 맵별로 다른 배경, 스폰 위치, 장애물, 파워업 후보 지점
- 플레이어와 봇이 함께 싸우는 팀 기반 전투
- 이동, 조준, 발사, 피격, 체력, 리스폰, 팀 점수 시스템
- 팀 규모에 따라 동적으로 계산되는 선승 점수 규칙
- 파워업 3종: 회복, 연사 강화, 1회 보호막
- 아이템이 장애물 위가 아니라 실제로 탱크가 도달 가능한 위치에만 생성되도록 보정
- 경기 종료 화면, 재시작, 로비 복귀
- 일시정지 오버레이: 이어하기, 재시작, 로비 복귀 지원
- 전체화면 전환 (`F`)
- 디버그/검증 인터페이스: `window.__gameDebug`, `window.render_game_to_text`, Playwright 검증 스크립트
- 사운드 에셋 팩 포함: `sound/`에 BGM/효과음 파일이 준비되어 있으며, 재생 로직은 아직 연결 전

## 현재 게임 흐름

1. 로비에서 맵과 `N vs M` 구성을 선택합니다.
2. 전투가 시작되면 플레이어와 봇이 팀 단위로 교전합니다.
3. 팀 킬 수가 목표 점수에 먼저 도달하면 승리합니다.
4. 전투 중 파워업은 맵 위의 이동 가능한 위치에서만 생성됩니다.

## 실행 방법

```bash
npm install
npm start
```

브라우저에서 [http://127.0.0.1:4173](http://127.0.0.1:4173) 으로 접속하면 플레이할 수 있습니다.

## 자동 검증

검증 스크립트는 실행 중인 로컬 서버를 기준으로 동작합니다.

```bash
# 터미널 1
npm start

# 터미널 2
npm run verify
```

기본 포트 `4173`이 이미 사용 중이면 다른 포트로도 실행할 수 있습니다.

```bash
# 터미널 1
PORT=4174 node scripts/static-server.js

# 터미널 2
PORT=4174 node scripts/verify-game.js
```

검증 범위:

- 메뉴 진입
- `N vs M` 팀 수 반영
- 파워업 적용
- 팀 점수 및 경기 종료
- 맵 선택 반영
- 맵별 지형 노출
- 파워업 스폰 위치의 충돌/도달 가능 여부

## 조작법

- 이동: `WASD` 또는 방향키
- 조준: 마우스
- 발사: 마우스 클릭 또는 `Space`
- 팀 수 조절: 메뉴의 `+`, `-` 버튼
- 맵 변경: 메뉴의 좌우 화살표 버튼
- 일시정지: `P` 또는 `Esc`
- 전체화면: `F`

## 프로젝트 구조

- `index.html`: 게임 캔버스 진입점
- `styles.css`: UI 및 레이아웃 스타일
- `app.js`: 게임 상태, 렌더링, 입력, AI, 파워업, 디버그 로직
- `scripts/static-server.js`: 로컬 정적 서버
- `scripts/verify-game.js`: Playwright 기반 자동 검증
- `sound/`: BGM/효과음 에셋과 생성 스크립트

## 로드맵

- 사운드 재생 로직 연결 및 볼륨/ON-OFF 설정
- 난이도 조정
- 모바일 입력 지원
- 맵 추가 및 맵별 규칙 확장
- 팀 AI 고도화
- 설치형 무기(예: 지뢰) 같은 전략 요소 추가
- 장기적으로 멀티플레이 구조 검토

## 협업 최소 규칙

- 한 번에 하나의 목적만 수정한다. 기능 추가, 버그 수정, 리팩터링을 한 커밋에 섞지 않는다.
- 작업 전 현재 브랜치와 변경 파일을 확인한다.
- 동작이 바뀌면 직접 실행해서 확인하고, 가능하면 `npm run verify`까지 통과시킨다.
- 큰 구조 변경 전에는 README나 이슈에 의도를 먼저 남긴다.
- 사용하지 않는 코드나 임시 로그는 커밋 전에 제거한다.
- `node_modules`, 테스트 산출물, 개인 로컬 파일은 커밋하지 않는다.
- 새 기능을 넣으면 README의 기능 설명이나 명세도 같이 갱신한다.
- 다른 사람이 작업 중인 파일은 함부로 덮어쓰지 말고, 충돌 가능성이 있으면 먼저 공유한다.
